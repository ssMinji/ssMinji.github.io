#### 단어 뒤집기
---

모든 단어가 역순으로 들어있는 메시지를 해독하기 위해 단어 순서를 뒤집어야 한다고 해보자. 문자 리스트를 받아 단어의 순서를 뒤집는 함수인 ```reverse_words()```를 작성해보자. 

> 왜 문자열 대신에 문자 리스트일까? 
>
> 이 문제의 목표는 문자열을 in place로 조작하는 것을 연습하는 것이다. 메시지를 수정하려고 하고 있기 때문에, immutable한 문자열 대신 리스트같은 mutable한 타입이 필요하다. 

예를 들어:

```python
message = [ 'c', 'a', 'k', 'e', ' ',
            'p', 'o', 'u', 'n', 'd', ' ',
            's', 't', 'e', 'a', 'l' ]

reverse_words(message)

print(''.join(message))
# 'steal pound cake'
```

메시지는 오직 문자와 공백만을 포함하고, 모든 단어들은 하나의 공백으로 나뉘어진다고 가정한다. 



#### Gotchas
---

우리는 이것을 O(1) 공간으로 해결할 수 있다. ***in-place***를 이용할 것임을 기억하자. 

그리고 O(n) 시간으로 풀 수 있다. 

각각의 단어들을 한 번에 하나씩 바꾼다고 했을 때, 단어들의 길이가 다를 경우에 어떨지 생각해보자. 최악의 경우 각 swap은 O(n) 시간이 소요될 것이다. 



#### Breakdown
---

더 간단한 문제에서 출발해보자. 만약 메시지에 있는 모든 문자들을 역순으로 배열하려면 어떻게 해야 할까? 

그렇다면, 첫 번째 문자와 마지막 문자를 서로 바꾸고, 그 다음 두 번째 문자와 마지막에서 두 번째 단어를 서로 바꾸고, 중간에 도달할 때까지 그런 식으로 바꾸면 될 것이다. 이를 코드로 구현해보자. 

```python
def reverse_characters(message):
    left_index = 0
    right_index - len(message) - 1
    
    # 양쪽 끝단에서부터 중간에 도달할 때까지 
    while left_index < right_index:
        # 왼쪽 단어와 오른쪽 단어를 교환한다
        message[left_index], message[right_index] = message[right_index], message[left_index]
        left_index += 1
        right_index -= 1
```

문자를 서로 교환하기 위해 한 줄의 간단한 코드로 작성했다. 만약 다른 언어를 사용한다면, 한 변수를 ```temp``` 변수에 저장해 직접 바꿔줘야 할 것이다. 

그럼 여기까지 했을 때, **이 작업이 우리를 도와주는가?**

같은 컨셉을 이용하지만 이를 *단어* 대신 *문자*에 적용할 수 있을까?

아마도 그럴 것이다. 우리는 몇 가지 사항을 알아내야 할 필요가 있다. 

1. 단어의 시작점과 끝지점을 어떻게 알아낼 것인가?
2. 두 단어들의 시작점과 끝지점을 안다고 했을 때, 그 두 단어를 어떻게 서로 교환할 것인가?

둘 중 아무거나 먼저 해결해도 되지만, 해보지 않아도 런타임에서 잠재적으로 문제가 생길 것이 보인다. 무엇이 문제인지 짐작이 가는가?

길이가 *다른* 두 단어를 서로 교환할 때 무슨 일이 벌어질까?

```python
# the eagle has landed
[ 't', 'h', 'e', ' ', 'e', 'a', 'g', 'l', 'e', ' ',
  'h', 'a', 's', ' ', 'l', 'a', 'n', 'd', 'e', 'd' ]
```

 이미 'the'와 'landed'의 시작점과 끝지점을 안다고 가정해보자. 이 둘을 서로 교환하려면 얼마나 걸릴까? 

n이 메시지의 총 길이일 때, O(n) 시간이 걸린다! 

왜일까? 'the'를 뒤로 보내고 'landed'를 앞으로 보내기 위해서, 'landed'가 'the' 보다 더 길기 때문에, 그 두 단어 사이에 있는 모든 것을 옮겨야 한다. 

따라서 이 접근방식에서의 총 시간 비용은 어떻게 될까? 한 번 훑음으로써 모든 단어들의 시작점과 끝지점을 알 수 있다고 가정해보자. (O(n) 시간)

총 O(n^2) 시간이 걸릴 것이다. 왜일까? 최악의 경우 문자 개수만큼의 단어가 존재할 수 있을 것이고, 다른 길이를 가진 단어들을 서로 교환해야 할 것이기 때문이다. 예를 들면:

```python
# a bb c dd e ff g hh
[ 'a', ' ', 'b', 'b', ' ', 'c', ' ', 'd', 'd', ' ',
  'e', ' ', 'f', 'f', ' ', 'g', ' ', 'h', 'h' ]
```

첫 번째 단어와 마지막 단어를 서로 교환하기 위해 O(n) 시간이 든다. (모든 n개의 문자를 이동시켜야 한다.)

```python
# Input: a bb c dd e ff g hh
[ 'a', ' ', 'b', 'b', ' ', 'c', ' ', 'd', 'd', ' ',
  'e', ' ', 'f', 'f', ' ', 'g', ' ', 'h', 'h' ]

# First swap: hh bb c dd e ff g a
[ 'h', 'h', ' ', 'b', 'b', ' ', 'c', ' ', 'd', 'd',
  ' ', 'e', ' ', 'f', 'f', ' ', 'g', ' ', 'a' ]
```

그리고나서 두 번째 교환에서는:

```python
# Input: a bb c dd e ff g hh
[ 'a', ' ', 'b', 'b', ' ', 'c', ' ', 'd', 'd', ' ',
  'e', ' ', 'f', 'f', ' ', 'g', ' ', 'h', 'h' ]

# First swap: hh bb c dd e ff g a
[ 'h', 'h', ' ', 'b', 'b', ' ', 'c', ' ', 'd', 'd',
  ' ', 'e', ' ', 'f', 'f', ' ', 'g', ' ', 'a' ]

# Second swap: hh g c dd e ff bb a
[ 'h', 'h', ' ', 'g', ' ', 'c', ' ', 'd', 'd',
  ' ', 'e', ' ', 'f', 'f', ' ', 'b', 'b', ' ', 'a' ]
```

첫 번째와 마지막 단어, 그리고 몇 개의 공백을 *제외한* 모든 n개의 문자를 이동시켜야 한다. 따라서 총 ```n-5```개의 문자를 이동시켜야 한다. 

3번째 교환에 대해서는, 이동시키지 않아도 되는 문자가 5개가 존재한다. 따라서 총 ```n-10```개의 문자를 이동시켜야 한다. 따라서 다음의 식에 도달하게 된다. 

​					```n + (n - 5) + (n - 10) + (n - 15) + ... + 6 + 1 ```

이는 일반적인 삼각배열의 하위 계열이다.

##### 삼각배열이란?

삼각배열이란 각 숫자가 정삼각형의 한 row를 이루는 숫자들의 나열을 의미한다. 

예를 들어 1, 2, 3, 4, 5는 숫자들을 다음과 같이 쌓을 수 있기 때문에 삼각배열이다. 

![](https://user-images.githubusercontent.com/44194800/78015274-29a86300-7384-11ea-9110-c04eaac306b5.PNG)

이 합은 15인데, 따라서 15를 삼각수라 한다. 

삼각배열은 항상 1부터 시작해 1씩 증가한다. 

삼각배열에서 변하는 유일한 것은 가장 큰 숫자의 값이기 때문에, 그것에 이름을 붙이는 것이 도움이 된다. 그것을 n이라 하자. 

```python
# n이 8
1, 2, 3, 4, 5, 6, 7, 8
```

삼각배열은 n이 얼마나 크든지 상관없이 모든 수들의 총 합을 구하는 것이 항상 간단하기 때문에 아주 좋다. 

바로 위의 예를 가져와보자. 첫번째 수와 마지막 수를 더하고, 두번째 수와 마지막에서 두번째 수를 더했을 때, 합이 같은 것을 알 수 있다! 이는 중간에 도달할 때까지의 모든 쌍에서 동일하다. 만약 모든 쌍들을 더한다면, 다음을 얻을 수 있을 것이다:

```python
1 + 8 = 9
2 + 7 = 9
3 + 6 = 9
4 + 5 = 9
```

다음의 사항은 *모든* 삼각배열에서 참이다. 

1. **양끝단의 숫자 쌍의 합은 항상 동일하다.** 
2. **그 합의 값은 배열의 n값보다 항상 1이 클 것이다.** 

따라서 패턴이 존재한다. 각 쌍의 합은 ```n+1```이고, ```n/2```개의 쌍이 존재한다. 따라서 총 합은 :

 따라서 위의 우리가 가진 식은 각 term 사이에 4 term을 건너뛰고 있을 뿐이다! 따라서 우리는 삼각배열에서 "매 5번째 수"의 합을 가지고 있는 것이다. 이는 총 합이 원래 배열의 합의 1/5 정도가 될 것이라는 것을 의미한다! 하지만 5로 나누는 것에 대한 빅 오 표기법은 상수이기 때문에, 무시할 수 있다. 원래 삼각배열은 O(n^2) 이므로, 매 5번째 요소에 대한 우리의 배열 또한 마찬가지가 된다! 

여기까지 이해가 되었다면, 결국 O(n^2) 시간이 걸림을 알 수 있다. 이것은 좋지 않다. 이것이 최선일 가능성도 있지만.....더 나은 방법으로 할 수 있을 수도 있지 않을까? 

손으로 예제 입력값을 조작해보자. 

그리고 문자-수준에서의 역순배열을 위해 했던 것을 기억해보자. 직접 해본다면,

```python
# Character-reversed: dednal sah elgae eht
```

뭔가를 눈치챘는가?

원하는 아웃풋와 대조해보면 어떤가?

```python
# Input: the eagle has landed
[ 't', 'h', 'e', ' ', 'e', 'a', 'g', 'l', 'e', ' ',
  'h', 'a', 's', ' ', 'l', 'a', 'n', 'd', 'e', 'd' ]

# Character-reversed: dednal sah elgae eht
[ 'd', 'e', 'd', 'n', 'a', 'l', ' ', 's', 'a', 'h', ' ',
  'e', 'l', 'g', 'a', 'e', ' ', 'e', 'h', 't' ]

[ 'l', 'a', 'n', 'd', 'e', 'd', ' ', 'h', 'a', 's', ' ',

```

문자의 역순배열은 단어를 뒤집는다! 이것은 훌륭한 첫 단계이다. 여기서부터 시작해, 우리는 각 단어의 순서를 제대로 맞추기만 하면 된다. 

더 정확히는, 각 단어를 re-reverse 해주면 된다! 

#### Solution
---

우리는 ```left_index```와 ```right_index``` 사이에 있는 모든 단어들을 바꿔주는 ```reverse_characters()```라는 helper 함수를 사용할 것이다. 다음의 목적으로 사용하게 되는데:

1. **전체 메시지에 있는 모든 단어들을 바꿔준다.** 올바른 단어의 순서를 뱉어주지만 각 단어는 역순이 된다. 
2. **각 단어의 문자를 거꾸로 배열한다.** 

``` python
def reverse_words(message):
    # 먼저, 전체 메시지에 있는 모든 문자를 역순으로 배열한다. 
    reverse_characters(message, 0, len(message)n-1)
    
    # 이는 올바를 단어 순서를 뱉어준다. 
    # 하지만 각 단어들은 거꾸로 되어있다. 
    
    # 이제 단어들의 문자들의 순서를 역순으로 나열함으로써
    # 제대로 돌려놓을 것이다
    
    # 현재 단어의 시작점을 저장해놓는다
    # 현재 단어의 끝지점을 찾아야 하기 때문이다
    current_word_start_index = 0
    
    for i in range(len(message) + 1):
        # 현재 단어의 끝지점을 찾는다
        if (i == len(message)) or (message[i] == ' '):
            reverse_characters(message, current_word_start_index, i - 1)
            # 만약 메시지가 끝나지 않았다면 
            # 다음 단어의 시작점은 한 문자 앞일 것이다 
            current_word_start_index = i + 1
            
def reverse_characters(message, left_index, right_index):
    # 양쪽 끝단에서부터 중간에 도달할 때까지 
    while left_index < right_index:
        # 왼쪽 단어와 오른쪽 단어를 교환한다
        message[left_index], message[right_index] = message[right_index], message[left_index]
        left_index += 1
        right_index -= 1
```

#### Complexity
---

O(n) 시간과 O(1) 공간

#### 결론
---

한 번에 하나씩 단어들의 순서를 바꾸는 naive한 해결방법은 최악의 경우 O(n^2) 런타임이 소요된다.  이것은 다른 길이를 가진 단어를 서로 바꾸기 위해서는 다른 모든 문자들을 이동시켜야 하기 때문이다. 

이 문제점을 해결하기 위해, 우선 메시지의 모든 문자들을 역순으로 나열시켰다. 이는 모든 단어들을 올바를 위치에 위치하게 하지만, 각 단어의 문자들은 순서가 거꾸로 되어있다.  따라서 최종 답을 얻기 위해, 각 단어에 있는 문자들을 역순으로 나열시켰다. 이는 메시지를 2번 pass하면 끝나게 되고, 따라서 총 O(n) 시간이 소요된다. 

**문제를 더 간단한 버전으로 쪼개서 풀어보고, 그것이 원래 문제를 푸는데 도움이 되는지를 보자. (우리가 풀어본 문제의 경우, 단어 대신 문자를 역순 나열하는 것이 될 것이다.)**
