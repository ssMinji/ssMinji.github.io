## 이진수(Binary number)

각 숫자는 *10*개의 값(1, 2, 3, 4, 5, 6, 7, 8, 9, 0)을 가지지 때문에, 우리가 사용하는 숫자 시스템은 십진수이다. 

하지만 컴퓨터는 열개의 값을 가진 숫자를 가지고 있지 않다. 컴퓨터는 두 개의 값(0과 1)의 값의 비트(bit)를 가지고 있다. 그래서 컴퓨터는 이진수를 사용한다. 

**십진법, 혹은 십진수**

이진수를 이해하기 위해서, 십진수가 어떻게 작동하는지 들여다보자. 

십진법에서의 수 "101"을 보면:

> 101

"1"이 두 번 나타나지만, 이 두 개의 "1"이 같은 것을 의미하지 않는다. 왼쪽 끝의 "1"은 100을, 오른쪽 끝의 "1"은 1을 의미한다. 그건 왼쪽끝의 "1"은 100의 자리에, 오른쪽 끝의 "1"은 1의 자리에 있기 때문이다. 가운데의 "0"은 10의 자리이다. 

>  100의 자리 | 10의 자리| 1의 자리
>
> ​         1                   0                 1
>
> ​      100         +       0          +     1
>
> = 101

각 자리수(1의 자리, 10의 자리, 100의 자리,...)는 10의 제곱근이다.:

> 10^0 = 1
>
> 10^1 = 10
>
> 10^2 = 100

**이진법, 혹은 이진수**

십진수의 자리수가 10의 제곱근이었던 것처럼, 이진수의 자리수도 2의 제곱근이다. 

> 2^0 = 1
>
> 2^1 = 2
>
> 2^2 = 4

그럼 이제, 같은 "101"을 이진법으로 읽어보자. 

> ​        1     |       0       |     1
>
> ​        4     +       0       +     1
>
> = 5

**이진법으로 10까지 세보기**

0부터 10까지 이진법으로 세는 방법과, 각 bit를 어떻게 해석하는지를 나타내보았다. 

| 십진법 | 이진법 | 해석          |
| ------ | ------ | ------------- |
| 0      | 0000   | 0 + 0 + 0 + 0 |
| 1      | 0001   | 0 + 0 + 0 + 1 |
| 2      | 0010   | 0 + 0 + 2 + 0 |
| 3      | 0011   | 0 + 0 + 2 + 1 |
| 4      | 0100   | 0 + 4 + 0 + 0 |
| 5      | 0101   | 0 + 4 + 0 + 1 |
| 6      | 0110   | 0 + 4 + 2 + 0 |
| 7      | 0111   | 0 + 4 + 2 + 1 |
| 8      | 1000   | 8 + 0 + 0 + 0 |
| 9      | 1001   | 8 + 0 + 0 + 1 |
| 10     | 1010   | 8 + 0 + 2 + 0 |

**음수와 2의 보수법**

이진법에서 **음수**는 일반적으로 2의 보수법을 사용해 나타낸다. 2의 보수법에서, 왼쪽끝 숫자는 *음수*를, 나머지 숫자는 양수를 나타낸다. 

그렇다면 "101"을 2의 보수법으로 해석하면 어떻게 되는지 알아보자: 

> ​      1     |     0       |    1
>
> ​     -4     +     0       +    1 
>
> = -3

**2의 보수법으로 -5부터 5까지 세보기**

 -5부터 5까지 2의 보수법으로 나타내는 방법과, 각 bit를 어떻게 해석하는지 나타내보았다. 

| 십진법 | 이진법 | 해석           |
| ------ | ------ | -------------- |
| -5     | 1011   | -8 + 0 + 2 + 1 |
| -4     | 1100   | -8 + 4 + 0 + 0 |
| -3     | 1101   | -8 + 4 + 0 + 1 |
| -2     | 1110   | -8 + 4 + 2+ 0  |
| -1     | 1111   | -8 + 4 + 2 + 1 |
| 0      | 0000   | 0 + 0 + 0 + 0  |
| 1      | 0001   | 0 + 0 + 0 + 1  |
| 2      | 0010   | 0 + 0 + 2 + 0  |
| 3      | 0011   | 0 + 0 + 2 + 1  |
| 4      | 0100   | 0 + 4 + 0 + 0  |
| 5      | 0101   | 0 + 4 + 0 + 1  |

> 그렇다면, 1011은 "11"(이진법)이라고 읽어야 할까 "-5"(2의 보수법)으로 읽어야 할까? 
>
> 둘 다 가능하다! 많은 언어들은 두 가지 타입의 수를 가지고 있다.(signed와 unsigned) Signed는 2의 보수법을 의미하고, unsigned는 단순 이진법을 의미한다.
>
> 꿀팁) 만약 면접관이 이진법을 십진법으로 변환하는 문제를 질문한다면, "이것이 단순 이진법인가요, 아니면 2의 보수법인가요?" 라고 물어보자!
